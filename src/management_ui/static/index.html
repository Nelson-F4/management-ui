Stock history

Unified timeline for one symbol: market_strategy_order_request, broker_order, and broker_position_history. Default view: rows whose event time falls in the last N hours (default 24). If any row in that window carries a correlation_id, all rows for that id are included even when older than the window. Rows without a correlation id only appear when in-window. Groups are ordered by newest activity; within a correlation, rows are newest first. Optional correlation id shows the full trace for that id (time window ignored). Each group is expand/collapse (click the header); the first group starts open, the rest start closed.

Open this tab and choose a symbol, then Load history.

RabbitMQ management

Opens the broker management plugin (port 15672 by default). Use the same URL your browser can reach (often http://localhost:15672). Set ui.public_rabbitmq_management_url on the Settings tab as JSON {"value":"http://host:15672"}, or set PUBLIC_RABBITMQ_MANAGEMENT_URL on control-plane and restart. If the frame is blank or blocked, use Open RabbitMQ.

Database settings

Non-secret keys only. Values are JSON objects (e.g. {"value": "..."}).

Message bus (topic)

bus.topology — exchange name, exchange_type (use topic; must match control-plane and execution-runtime), and queues map from logical consumer (extension key) to physical queue name. Each consumer binds its queue with the scheduler routing key for that task; the JSON body also includes a topic field for logging. Copy the JSON into NELSON_BUS_TOPOLOGY_JSON on execution-runtime (same object as here).
